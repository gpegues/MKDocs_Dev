# ServiceNow Request Submission Guidelines for Microsoft 365: Exchange Online Resource Access

## Introduction
This document provides detailed guidelines for submitting requests in ServiceNow for gaining access to various Microsoft 365 Exchange Online resources, including Distribution Groups, Dynamic Distribution Groups, Mail-Enabled Security Groups, and Microsoft 365 Groups. Adherence to these guidelines ensures efficient processing and accurate assignment of access.

## Request Submission Process

### General Instructions
- Access requests should be submitted through the ServiceNow platform.
    - Note: This request type via Service now is "Email, Communications and Shared Access Request"
    - Note: Access requests or inclusions submitted as incidents will be denied, as incidents are defined as troubleshooting events, and subverts any agency related approval process in the Service Now access request process.
- Clearly specify the type of resource access required: Distribution Group, Dynamic Distribution Group, Mail-Enabled Security Group, or Microsoft 365 Group.
- Provide a justification for the access request, detailing the need and purpose.

### User List Submission

#### Correct Submission Format
- **Use Proper Submission Fields**: When requesting access for multiple users, properly utilize the submission fields provided in the ServiceNow form.
- **Individual Entries**: Each user must be entered in the submission fields which pulls from the global address book, to ensure the correct person is granted access.
- **Please describe your request Field**: Please note, while we review information in this field, we will not process large amounts of data placed in this field or names of users. This field should not be used to attempt to avoid correctly inputing users into the following fields:
  - What is the full name of the distribution group
  - Select the owner(s) of the distribution group
  - Select members of the distribution group

#### Incorrect Submission Formats
- **Attached User Lists**: Do not attach spreadsheets or documents containing user lists.
- **Text Area Lists**: Avoid listing users in text area fields instead of using the submission fields.
  
### Reason for Submission Field Use
- **Accuracy in Identity Resolution**: Using submission fields helps to accurately identify individuals as their information is pulled from the Global Address Book. This is especially important in cases where people share the same name.
- **Efficient Processing**: Structured data entry ensures quicker processing and reduces the chances of errors or delays.
- **Data Consistency**: Helps in maintaining consistency and clarity of information across requests.

## Types of Access Requests

### Distribution Group Access
- Specify the name of the Distribution Group.
- Provide a list of users requiring access using the submission table.

### Dynamic Distribution Group Access
- Indicate the name of the Dynamic Distribution Group.
- Describe the criteria for dynamic inclusion in the group (e.g., department, role).
  - Please note, this is not a commonly requested group type.

### Mail-Enabled Security Group Access
- Mention the specific Mail-Enabled Security Group.
- Use the submission table to list individual users needing access.
  - Please note, this is not a commonly requested group type.

### Microsoft 365 Group Access
- Identify the Microsoft 365 Group in question.
- List users for access provision in the designated submission table.

## Resubmission Policy

### Instances Requiring Resubmission
- Submissions with attached user lists or text area lists instead of the provided submission fields.
- Incomplete, incorrect, or ambigous requests. 

### Importance of Adherence
- Ensures accurate and efficient processing of access requests.
- Ensures that data leakage of private resources are not unintentionally exposed to unintended individuals.
- Helps in avoiding confusion and potential security risks associated with incorrect access assignments.

## Why this is important

Adhering to these guidelines when submitting access requests in ServiceNow for Microsoft 365 Exchange Online resources is crucial for ensuring efficient processing and the correct assignment of access. Properly formatted requests help in maintaining data integrity, enhancing security, and ensuring that the right individuals gain access to the necessary resources.
